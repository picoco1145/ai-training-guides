# 클로드 스킬로 글쓰기 절차 자동화하기

> 🏷 태그: Claude · 자동화
> 👥 대상: 클로드 웹 초보자
> ⏱ 소요시간: 30분

## 💡 소개
클로드에는 **스킬(Skills)** 기능이 있습니다. 스킬은 "블로그 글을 쓸 때는 항상 이런 순서로 써라"는 절차를 미리 저장해두는 파일입니다. 한 번 만들어두면 "블로그 글 써줘"라고만 해도 클로드가 정해진 절차대로 글을 씁니다. 스킬 파일을 직접 작성할 필요 없이, 클로드에게 만들어달라고 하면 됩니다.

## 📦 사전 준비
- [claude.ai](https://claude.ai) **Pro 플랜** 계정 (스킬 업로드 기능은 Pro 전용입니다)
  - 무료 플랜 사용자: 스킬 파일을 만드는 것까지는 가능하나, 프로젝트에 업로드하려면 Pro 필요
- 과제 2에서 만든 글쓰기 스타일 분석 결과 (없어도 진행 가능)

---

## 📚 단계별 실습

### 1️⃣ 스킬 파일이 뭔지 이해하기

스킬 파일은 `.md` 확장자의 텍스트 파일입니다. 내부 구조는 단순합니다:

```
---
name: 블로그-글쓰기
description: 블로그 글 작성 요청 시 자동 실행. 키워드: 글 써줘, 블로그, 포스팅
---

# 블로그 글쓰기 절차

1. 주제를 받으면 먼저 후킹 제목 3개를 제안한다
2. 독자가 고른 제목으로 서론을 쓴다
3. 소제목 3~5개로 본문을 구성한다
4. 마지막에 CTA 문장으로 마무리한다
```

<!-- GENERATE: prompt="simple diagram showing skill file structure: YAML header at top, then workflow steps 1-2-3-4 as numbered boxes connected by arrows, clean flat design, white background" style=diagram -->

`name`과 `description`이 스킬의 이름과 발동 조건입니다. `description`에 쓴 키워드가 포함된 질문을 하면 이 스킬이 자동으로 실행됩니다.

✅ 확인: 위 구조를 이해했으면 다음 단계로 넘어가세요.

---

### 2️⃣ 클로드에게 스킬 파일 만들어달라고 요청하기

claude.ai에서 새 대화를 시작합니다. 아래 프롬프트를 입력합니다:

```
아래는 내 블로그 글쓰기 스타일이야.
이 스타일대로 글을 자동 작성해주는 클로드 스킬(SKILL.md) 파일을 만들어줘.

조건:
1. 맨 위에 YAML 메타데이터(name, description) 포함
2. description에는 "블로그", "글 써줘", "포스팅", "콘텐츠 작성" 키워드 포함
3. 내 글쓰기 특징: {여기에 과제2에서 나온 분석 결과 붙여넣기}
4. 주제만 입력하면 내 스타일대로 전체 글을 쓰는 절차를 단계별로 작성
5. 결과를 코드블록으로 감싸서 출력해줘
```

<!-- CAPTURE: url=https://claude.ai area=full annotate=highlight(.composer-parent,"프롬프트 입력창") -->

과제 2 결과가 없다면 {여기에 과제2에서 나온 분석 결과 붙여넣기} 부분을 아래처럼 바꿔도 됩니다:
```
- 친근한 말투, 독자에게 직접 말 거는 방식
- 실용적인 정보 위주, 어렵지 않게
- 단락 짧게, 불릿 포인트 활용
- 마지막에 "오늘 바로 해보세요" 같은 행동 유도 문장
```

✅ 확인: 클로드가 `---`로 시작하는 YAML 헤더가 포함된 코드블록을 출력하면 성공입니다.

---

### 3️⃣ 스킬 파일 저장하기

클로드가 출력한 코드블록 오른쪽 위의 **복사 버튼**을 클릭합니다.

<!-- CAPTURE: url=https://claude.ai area=full annotate=arrow(850,400,"코드블록 복사 버튼") -->

컴퓨터에 텍스트 편집기(메모장, TextEdit 등)를 열고 붙여넣기 합니다. 파일을 저장할 때 파일명을 반드시 `.md` 확장자로 저장합니다:
- `blog-skill.md`
- `글쓰기스킬.md`

✅ 확인: 파일 탐색기에서 `.md` 파일이 보이면 성공입니다.

---

### 4️⃣ 클로드 프로젝트에 스킬 업로드하기

과제 1에서 만든 프로젝트로 이동합니다. 프로젝트 설정 화면에서 **"Skills"** 또는 **"Add skill"** 섹션을 찾습니다.

<!-- CAPTURE: url=https://claude.ai area=full annotate=highlight([data-testid="project-skills"],"스킬 업로드 영역") -->

**"Upload skill"** 또는 **"+ Add"** 버튼을 클릭해 3단계에서 저장한 `.md` 파일을 선택합니다.

✅ 확인: 스킬 목록에 업로드한 파일 이름이 나타나면 성공입니다.

---

### 5️⃣ 스킬 자동 발동 테스트하기

프로젝트 안에서 새 대화를 시작합니다. 스킬 description에 넣었던 키워드를 포함해 간단히 입력합니다:

```
"AI 도구 비교 블로그 글 써줘"
```

<!-- CAPTURE: url=https://claude.ai area=full annotate=highlight(.streaming-response,"클로드 응답 영역") -->

클로드가 스킬에 정의된 절차대로:
1. 후킹 제목 3개 제안
2. 선택 후 서론 작성
3. 소제목으로 본문 구성
4. CTA로 마무리

순서로 진행하는지 확인합니다.

✅ 확인: 내가 아무 지시 없이 키워드만 입력했는데도 정해진 절차대로 글을 쓰기 시작하면 성공입니다.

---

## ⚠️ 자주 겪는 문제

- **스킬이 자동 발동이 안 된다**: 스킬 파일의 `description`에 쓴 키워드와 내가 입력한 문장의 단어가 일치해야 합니다. description에 "블로그"가 있는데 "포스팅 써줘"라고 하면 발동 안 됩니다. description에 "블로그, 포스팅, 글 써줘" 등 여러 키워드를 넣으세요.
- **파일을 업로드했는데 스킬 목록에 안 보인다**: 페이지를 새로고침(F5)하고 다시 확인하세요.
- **메모장에서 저장할 때 .md가 .txt로 바뀐다**: 파일 저장 시 파일 형식을 "모든 파일(\*.\*)"로 바꾸고 파일명을 `blog-skill.md`처럼 직접 입력하세요.

## 💼 실무 팁

- **스킬 description 키워드가 핵심**: description에 쓴 단어가 트리거입니다. 내가 자주 쓰는 표현을 넣으세요.
- **스킬은 여러 개 가능**: 블로그용, 이메일용, 보고서용 스킬을 따로 만들어 같은 프로젝트에 올릴 수 있습니다.
- **스킬 수정도 쉽습니다**: .md 파일을 수정해서 다시 업로드하면 됩니다.
