# 콘텐츠 품질 규칙

## 초보자 기준
- 전문 용어 첫 등장 시 괄호 설명 필수
- 클릭 경로는 "메뉴 > 하위메뉴 > 버튼" 형태로 구체적 명시
- 각 단계 끝에 "이렇게 되면 성공" 확인 포인트 필수
- 흔한 실수/에러에 대한 트러블슈팅 포함

## 구조 규칙
- 모든 가이드에 최소 2개 이상의 실습 단계
- 각 단계에 최소 1장의 이미지 지시서
- 소개 섹션에 "왜 이 과제를 하는지" 2~3문장

## 이미지 품질
- 텍스트 가독성 확보 (폰트 크기, 대비)
- 어노테이션(화살표, 하이라이트) 명확
- 한국어 UI 텍스트 정확

## 금지 사항
- Gemini로 소프트웨어 UI 스크린샷 생성 금지 (Track A 캡처만)
- 이미지 없는 단계 금지 (최소 CAPTURE 또는 GENERATE 지시서 포함)
- "자세한 내용은 검색하세요" 같은 떠넘기기 금지
